Debug 是程序员十分重要的技能，比如我们自己写的程序出现异常，我们可以通过Debug模式去分析异常。

再者，现如今开源代码层出不穷，作者最近正好在分析开源代码。Debug在这里起到了很重要的作用。面对复杂的代码，我们要找到相对的执行流程、执行方法不太容易。

通过Debug模式，我们可以跟踪程序的执行流程，这样，某项业务的执行步骤便一目了然了。

1 基本用法

本文使用环境为 Intellij IDEA ，但其它的IDE与此大同小异。

1.1 组件1

这组组件共八个按钮，从左至右编号1、2、3、4、5、6、7、8。

###zujian1###

  1. Show Execution Point：点击此按钮可将光标定位到当前代码执行处；
  2. Step Over：步过，一行一行往下执行，如果该行上有方法，不会进入方法；
  3. Step Into：步入，一行一行往下执行，如果该行上有方法，进入方法内部执行，当一般进入自定义方法，不会进入官方类库的方法；
  4. Force Step Into：强制步入，能进入任何方法，查看底层源码是可通过这个进入官方类库的方法；
  5. Step Out：步出，从步入的方法内退出到方法调用处；
  6. Drop Frame：回退断点，默认无；
  7. Run to Cursor：运行至光标处，将光标定位到需要查看的那一行，代码会运行至光标行，无需打断点；
  8. Evaluate Expression：计算表达式。

1.2 组件2

这组组件共七个按钮，从上至下编号1、2、3、4、5、6、7。

###zujian2###

  1. Rerun ‘xxxx’：重新运行程序，会关闭服务后重新启动程序；
  2. Update：更新程序，在代码改动后可执行这个功能来应用更新；
  3. Resume Program：恢复程序，点击该按钮将运行至下一个断点。比如我们在程序设置在代码第10行设置一个断点，在代码第20行设置一个断点，现在但马运行到第10行，点击按钮，代码将运行至下一个断点（第20行），若后面无断点，则运行完整个流程；
  4. Pause Program：暂停程序，启用Debug；
  5. Stop：连续按两下，关闭程序；
  6. View Breakpoints：查看所有断点；
  7. Mute Breakpoints：消除断点，点击该按钮将消除所有断点，断点将变为灰色，断点失效；再次点击，断点将变为红色，断点有效；如果只想对单个断点执行操作，直接在断点上右键选择相应的操作；

2 结语

通过基本操作我们已经可以初步的Debug我们的程序，来跟踪程序的执行。当然，Debug模式还有更多强大的功能，比如变量查看、智能步入、多线程调试等。下面这篇文章对Debug模式进行了详细的描述，推荐给大家。
